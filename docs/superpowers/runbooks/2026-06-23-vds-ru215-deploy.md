## Step 0 — specs + swap (this box may be 1 GB RAM)

```bash
ssh vdska2 'nproc; free -h; df -h /; lsb_release -d'
```
If RAM < 2 GB (likely), add swap BEFORE any docker build or the build will OOM:
```bash
ssh vdska2 'fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile && echo "/swapfile none swap sw 0 0" >> /etc/fstab && free -h'
```
> If `pnpm build` (frontend) still OOMs even with swap, fall back to building the
> image on your laptop and shipping it: `docker build ... -t lia-frontend . && docker save lia-frontend | ssh vdska2 'docker load'` — then skip the on-box build in Step 6.

## Step 6 — build + run the frontend pointed at the real API

```bash
ssh vdska2 'cd /opt/lia/frontend && docker build --build-arg NEXT_PUBLIC_API_URL=https://api.lia.pashteto.com -t lia-frontend . && docker rm -f lia-frontend 2>/dev/null; docker run -d --restart unless-stopped --name lia-frontend -p 127.0.0.1:3001:3001 lia-frontend'
```

## Step 7 — nginx vhosts (HTTP first, no TLS yet)

Create `/etc/nginx/sites-available/lia` on the box:
```nginx
server {
  listen 80; server_name lia.pashteto.com;
  location / { proxy_pass http://127.0.0.1:3001; proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme; }
}
server {
  listen 80; server_name api.lia.pashteto.com;
  location / { proxy_pass http://127.0.0.1:9080; proxy_set_header Host $host; proxy_set_header X-Forwarded-Proto $scheme; }
}
```
```bash
ssh vdska2 'ln -sf /etc/nginx/sites-available/lia /etc/nginx/sites-enabled/lia && nginx -t && systemctl reload nginx'
```
